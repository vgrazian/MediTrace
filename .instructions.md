# MediTrace — Agent Instructions

## Project Identity

- **Name**: MediTrace
- **Type**: Offline-first installable PWA for medication management in care homes
- **Repo**: `vgrazian/MediTrace` (GitHub Pages: `vgrazian.github.io/MediTrace/`)
- **Branch**: `gh-pages` (deployment), `main` (source of truth)
- **Credentials**: `credentials.local.env` (gitignored)

## Stack

| Layer | Technology |
|---|---|
| Framework | Vue 3 (Composition API, `<script setup>`) |
| Build | Vite 5 |
| Routing | Vue Router (hash mode: `createWebHashHistory`) |
| Styling | **Vanilla CSS** — NO Tailwind, NO preprocessors |
| Database | Dexie.js (IndexedDB wrapper) |
| Auth | Supabase (`@supabase/supabase-js`) + local fallback |
| Sync | Supabase realtime + Google Drive appDataFolder |
| PWA | `vite-plugin-pwa` (generateSW, autoUpdate) |
| Testing | Vitest (unit), Playwright (e2e) |

- `font-variant-numeric: tabular-nums` on `.conflict-table td`

### Component Conventions

- Cards: `.card` — border + shadow + `.75rem` radius
- Buttons: `.btn`, `.btn-primary`, `.btn-danger`, `.btn-ghost`, `.btn-sm`
- All interactive elements MUST have `:hover` AND `:active` states
- Loading states: `.loading-skeleton` with pulse animation (never plain "Caricamento..." text)
- Error messages: `.import-error` class with `role="alert"` attribute
- Empty states: guided messages with keyboard shortcut hints (e.g., "Premi N o clicca Aggiungi...")
- No emoji in UI text — use CSS badges/indicators instead

### Accessibility

- Skip-to-content link (`.skip-link`) on every page
- All interactive elements must have visible `:focus-visible` rings
- Error messages must have `role="alert"`
- Loading states must have `role="status"` with `aria-label`

## Deployment Workflow

**CRITICAL**: GitHub Pages serves from the branch ROOT, not `pwa/dist/`. After every build, sync:

```bash
# Always build with the correct base path:
cd pwa && VITE_BASE_URL=/MediTrace/ npx vite build

# Sync to root:
cp pwa/dist/index.html index.html
cp pwa/dist/manifest.webmanifest manifest.webmanifest
cp pwa/dist/sw.js sw.js
cp pwa/dist/favicon.svg favicon.svg
cp -r pwa/dist/assets/* assets/
cp -r pwa/dist/icons/* icons/

# Or use the deploy script:
bash scripts/deploy-gh-pages.sh
```

**CDN cache**: GitHub Pages CDN has `max-age=600` (10 min). After push, wait 10 min before verifying.

## Common Pitfalls

1. **Never `npx vite build` without `VITE_BASE_URL=/MediTrace/`** — produces wrong asset paths (`/assets/...` instead of `/MediTrace/assets/...`), causing white screen.
2. **Always sync `pwa/dist/*` to branch root** — GitHub Pages serves from root, not `pwa/dist/`.
3. **Service workers cache aggressively** — after deploy, click "Aggiorna app" or wait for SW update cycle.
4. **`confirm()` is banned** — use `openConfirmDialog()` from `../services/confirmDialog`.
5. **Don't use `import.meta.env` directly in templates** — extract to `const` in `<script setup>`.
6. **Android Go constraint**: no heavy animations, no GSAP, no large images. Keep bundle < 2MB.
7. **Demo data lives in `services/seedData.js`** — load via Impostazioni UI button.

## Naming Conventions

- Components: PascalCase (`AppNav.vue`, `CrudFilterBar.vue`)
- Composables: `useXxx` (`useAuth`, `useSessionViewState`)
- Services: camelCase (`buildReminderRows`, `markReminder`)
- CSS classes: kebab-case (`.conflict-table`, `.reminder-state`)
- Italian UI labels: Aggiungi/Modifica/Elimina (not Add/Edit/Delete)
